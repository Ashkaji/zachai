# Epic 2 Retrospective — Gestion Dynamique des Projets

**Project:** zachai  
**Date:** 2026-03-30  
**Epic goal:** Dynamic natures/labels, Label Studio provisioning via Camunda, audio upload + FFmpeg normalization, assignment dashboard (F08, F09, F10, F13).  
**Sprint-status:** Stories **2-1** … **2-4** → `done`.

---

## 1. Delivery summary

| Story | Delivered capability |
|-------|---------------------|
| **2.1** | Nature + label schema CRUD, Label Studio XML generation, async SQLAlchemy |
| **2.2** | Project creation, Camunda-driven Label Studio provisioning, project/audio ORM |
| **2.3** | Project-scoped audio upload, presigned flow, FFmpeg normalize registration, schema extensions |
| **2.4** | Manager project status / aggregates, assign transcripteur, transcripteur task list, status semantics fix post-FFmpeg |

Cross-cutting: **RBAC** (`get_roles`, Manager/Admin/Transcripteur), **flat JSON errors**, **mock_db** test pattern matured across stories.

---

## 2. What went well

- **2.1 code reviews** closed many concrete issues (XML injection, DB URL encoding, transaction boundaries, duplicate labels) — raised quality bar for `main.py` growth.
- **2.2** explicitly documented **eventual consistency** (Camunda vs committed project) — sets expectations for ops and QA.
- **2.4** clarified **status semantics** (`uploaded` vs `transcribed` after normalize) — avoids downstream workflow bugs.
- **Review-added tests** (409 races, 404 cross-project audio, Admin debug query) improved API predictability.

**Alice (Product Owner):** “Epic 2 is where the product starts feeling like ‘projects’ not just ‘infra.’”

**Charlie (Senior Dev):** “Camunda + FastAPI + DB in one epic — we still kept stories shippable.”

---

## 3. Challenges & deferred debt

| Theme | Evidence |
|--------|-----------|
| **Schema evolution** | Alembic missing — repeated in 2.1 deferrals and 4.x deferrals |
| **Auth churn** | Duplicated role checks; legacy `PUT .../status` without owner parity (2.4 defer) |
| **API surface drift** | Legacy `POST /v1/upload/request-put` vs project-scoped upload (2.3 defer) |
| **Distributed timing** | JWKS tolerance, Camunda start vs DB commit (2.2 defer — accepted model) |
| **Worker DB** | asyncpg per-call in worker (2.2 defer) |
| **N+1** | `list_natures` pattern (2.1 defer) |

## 6. Key takeaways

1. **Explicit consistency models** in story files prevented false “bugs” in Camunda delays.
2. **Status enum semantics** are product contracts — worth a one-pager when Epic 6 extends them.
3. **Defer API unification** only with a visible backlog ID — otherwise new features amplify drift.

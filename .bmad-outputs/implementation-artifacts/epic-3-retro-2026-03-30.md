# Epic 3 Retrospective — Pipeline d'Inférence Haute Performance

**Project:** zachai  
**Date:** 2026-03-30  
**Epic goal:** Normalized audio path, OpenVINO Whisper pre-annotation, model registry + hot-reload (F03, F04, F06).  
**Sprint-status:** Stories **3-1**, **3-2**, **3-3** → `done`.

---

## 1. Delivery summary

| Story | Delivered capability |
|-------|---------------------|
| **3.1** | FFmpeg worker: normalize to 16 kHz mono PCM, batch/realtime paths, MinIO I/O, env validation, subprocess timeouts |
| **3.2** | OpenVINO worker: `/transcribe`, segment JSON contract, threading + timeout pattern, WAV validation, confidence handling fixes |
| **3.3** | Poll `models/latest`, sync tree from MinIO, hot-reload `WhisperPipeline`, `/health` registry fields, smoke guidance |

**Engineering theme:** Heavy **worker** logic (blocking native code, MinIO, env guards) with **FastAPI** as orchestrator consumer in later epics.

---

## 2. What went well

- **3.1 review** drove production hygiene: input size limits, `anyio.to_thread` for FFmpeg, subprocess timeouts, startup env validation, orphan temp cleanup.
- **3.2 review** caught **confidence math**, **object size guards**, **test mode footgun**, **dependency pinning** — typical of ML serving risk.
- **3.3** aligned **registry pointer semantics** with architecture (`models/latest` text body) and documented lock/health tradeoffs.
- **Internal shield** maintained: worker not exposed on host per architecture §5.

**Dana (QA Engineer):** “Reviews on 3.2/3.3 read like a checklist for ‘how not to leak GPUs and disk.’”

---

## 3. Challenges & deferred debt

| Item | Notes |
|------|--------|
| **504 vs running inference** | `anyio.fail_after` returns while thread may still run — deferred, documented v1 limitation |
| **TOCTOU MinIO** | stat → get race; rare, deferred |
| **`retired_engines` growth** | Long reload loops — defer cap/eviction |
| **Global `model_lock`** | Serializes transcribe vs reload — intentional; ops must expect latency on swap |
| **IR manifest validation** | Indirect via `WhisperPipeline.load`; explicit manifest optional |

## 6. Key takeaways

1. **Worker stories need the same review bar as API stories** — env parsing, timeouts, and test-mode guards are security/reliability issues.
2. **Hot-reload correctness** is a concurrency story; locks and health snapshots must be documented for SRE.
3. **Deferred runtime limitations** (504/thread) should stay visible until Epic 7 or scale testing forces a redesign.

## 2. What went well (systems & process)

- **Single ingest core:** `persist_golden_set_entry` and shared counter semantics gave a stable contract for both expert and user loops (4.1 → 4.2).
- **Explicit normalization layer:** Label Studio payload handling via `golden_set.py` adapters reduced “mystery JSON” risk and made review feedback actionable (4.1).
- **Code review loop:** Stories 4.1–4.4 accumulated substantive review patches (idempotency, validators, MinIO hygiene, Camunda failure handling); most items were cleared or consciously deferred with owners in `deferred-work.md`.
- **Vertical slice acceptance:** 4.4 Dev Notes explicitly allowed slicing (BPMN + stub workers first); team kept `main` green while expanding scope.
- **MVP honesty:** Production `lora-training` limitation documented in worker README after review decision — avoids silent false confidence.

**Bob (Scrum Master):** “The epic’s strength is the same as its risk: everything touches Golden Set + Camunda. We still shipped coherent increments.”

**Alice (Product Owner):** “We have two real feedback loops into training — expert and transcripteur — that match the PRD narrative.”

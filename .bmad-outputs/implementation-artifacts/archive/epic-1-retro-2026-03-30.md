# Epic 1 Retrospective — Socle Infrastructure & Identité

**Project:** zachai  
**Date:** 2026-03-30  
**Epic goal:** Zero-trust foundations, sovereign storage, role model (F01, F02, F03-partie normalisation).  
**Sprint-status:** Stories **1-1**, **1-2**, **1-3** → `done` (all tracked stories complete).

---

## 1. Delivery summary

| Story | Delivered capability |
|-------|---------------------|
| **1.1** | MinIO bootstrap — four buckets (`projects`, `golden-set`, `models`, `snapshots`), init + healthchecks, compose skeleton for later services |
| **1.2** | PostgreSQL + Keycloak, realm `zachai-realm.json`, multi-role model (Admin / Manager / Transcripteur / Expert) |
| **1.3** | FastAPI gateway: JWKS JWT verify, presigned PUT/GET to MinIO, dual MinIO client pattern, first API tests |

**Architectural win:** Story 1.1 Dev Notes anchored **exact bucket names and startup order** for the whole stack; later epics consistently referenced that contract.

---

## 2. What went well

- **Review discipline early:** 1.1 patches (shell guards, quoted env vars, healthcheck path) hardened ops defaults before scale.
- **1.3 integration clarity:** Completion notes document the real bug class (double `projects/` prefix) and fix — good template for future MinIO key design.
- **Keycloak + Keycloak 26 profile flow** captured in 1.3 completion notes (VERIFY_PROFILE) — reduces rediscovery cost for new devs.
- **Tests from day one:** 1.3 shipped with mocked JWKS + MinIO and smoke path documented.

**Bob (Scrum Master):** “Epic 1 is ‘boring’ in the right way — storage, identity, and upload plumbing without business UI.”

## 3. Challenges & deferred debt (systems, not people)

| Item | Source | Note |
|------|--------|------|
| Shared Postgres superuser for multiple services | `deferred-work.md` (1-2 review) | Least-privilege split deferred; affects hardening backlog |
| No container CPU/mem limits | `deferred-work.md` (1-2 review) | Keycloak JVM spikes called out |
| Tolerant JWKS at startup | Echoed in Epic 2.2 deferred | FastAPI can start without Keycloak; intentional but worth monitoring |
| Presigned vs internal MinIO hostname | 1.3 dual-client pattern | Correct but easy to misconfigure in `.env` — docs are the safety net |

## 6. Key takeaways

1. **Foundation docs in story files** (1.1 bucket table) paid dividends across Epic 2–4.
2. **Identity edge cases** (Keycloak versions, admin port collisions) belong in runbooks early.
3. **Defer least-privilege DB** consciously — track it or it compounds.

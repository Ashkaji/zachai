### What went well

- Vertical sequencing was effective: backend IAM foundations first, then UI flows.
- Existing API/UI patterns were reused (`createUser`, dashboard modals), limiting churn.
- Review loops were productive: 16.5/16.6 tightened UX safety, trimming, error handling, and link guards.
- Test discipline stayed strong across API and frontend; final verification runs remained green.
- Story documentation quality improved mid-epic (clearer ACs and explicit technical guardrails).

### Challenges and system insights

- Keycloak + DB split-transaction remains a known trade-off (compensation cleanup is present, but strict distributed atomicity is still deferred by design).
- Story artifacts needed retroactive clarification in places (notably around Expert semantics and UI messaging), which was corrected before close.
- Role semantics were initially ambiguous in product language (`Expert` vs `Transcripteur` overlap). This is now explicitly documented and implemented as composite IAM roles.

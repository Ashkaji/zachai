## 3. What went well

- **Security-first handshake path:** Single-use WSS ticket flow held up and integrated cleanly with collaboration runtime.
- **Non-mutating UI design discipline:** Karaoke and grammar overlays stayed decoration/state-based, avoiding unintended Yjs churn.
- **Review-driven hardening loop:** Multiple patch cycles converged on robust behavior rather than cosmetic completion.
- **Cross-story contract reuse:** Access gates and role semantics stayed aligned across ticketing, media access, and grammar proxy.
- **Operational docs parity:** API mapping, compose env wiring, and story artifacts remained synchronized with implementation state.

---

## 4. Recurring challenges and technical debt

| Theme | Evidence in Epic 5 | Debt / risk signal |
|-------|---------------------|--------------------|
| **Race conditions under concurrency** | Redis lock ownership and follower coalescing refinements in 5.5 | Future async endpoints need ownership-safe lock helpers by default |
| **Stale state in rich editor UI** | apply-time recompute and stale-generation guards in grammar flow | Shared utility patterns for "stale response invalidation" should be standardized |
| **Compose readiness trade-offs** | LanguageTool dependency and health behavior tuning | Service startup policies should be documented per critical/non-critical dependency |
| **Validation at boundaries** | Upstream malformed payload handling and cache payload schema checks | Boundary validation checklist should be mandatory for proxy endpoints |
| **Large-scope patch cadence** | Multiple re-reviews to close high/medium findings | Earlier adversarial checks can reduce late-stage stabilization churn |

---

## 5. Quality and testing lessons

- **Tests that mattered most:** lock token release safety, cache corruption recovery, stale generation invalidation, offset/range mapping safety.
- **Pattern confirmed:** deterministic unit tests around edge contracts prevented regressions better than broad late-stage exploratory runs alone.
- **Gap to improve:** dedicated stress/concurrency harness for collaboration + grammar proxy interactions is still limited.
- **Practical lesson:** include explicit tests for degraded and malformed upstream responses whenever introducing a proxy route.
